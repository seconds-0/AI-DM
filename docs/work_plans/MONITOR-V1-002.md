## Problem Statement

Implement basic hooks within the application logic to facilitate cost tracking and analysis. This primarily involves logging key events related to potentially expensive API calls (Gemini, Live Voice API) and ensuring GCP Billing alerts are configured.

## Components Involved

*   `src/flows/storyteller.flow.ts`
*   `src/flows/planner.flow.ts`
*   `src/discord/liveVoiceHandler.ts` (Potentially)
*   `src/utils/logger.ts` (Structured logger)
*   Google Cloud Billing (Alerts)
*   Google Cloud Logging (Where usage logs go)

## Proposed Solution / Design Approach

1.  **Log Key API Calls:** Add structured log entries specifically indicating when calls are made to high-cost APIs:
    *   **Gemini (`generate`):** Log before and after the `generate` call in both Storyteller and Planner flows. Include the model name used and potentially token counts if easily available from the response (Genkit might provide this).
    *   **Live Voice API:** Log the start and end of a Live Voice API streaming session within the Discord Bot (`liveVoiceHandler.ts`). Include session duration if possible.
2.  **Standardize Log Format:** Ensure these specific cost-related log entries have a consistent format or marker (e.g., a specific `logType: 'cost_event'` field) to allow easier filtering and aggregation in Cloud Logging for analysis.
3.  **Billing Alerts:** Confirm that GCP Billing budget alerts are configured manually in the GCP Console for the project(s) to notify stakeholders if costs exceed predefined thresholds.
4.  **Session Cost Estimation (Deferred):** The PRD mentions storing estimated cost in `session_log` metadata. For V1, explicitly defer the *calculation* and *storage* of this estimate. Focus only on logging the raw usage events.
